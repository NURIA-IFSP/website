How to Use This Platform

LabViR is a virtual learning platform dedicated to robotics education. Here, you will find interactive courses that cover everything from the fundamentals of robotics to advanced applications using ROS (Robot Operating System).

The platform uses the concept of containers to run the courses. Containers are Docker images that contain all the resources needed to run the courses. They can be downloaded and run locally or in the cloud through a web browser. In this project, we chose GitHub Codespaces as the virtual execution environment. Its use is free for GitHub users and has a limit of 60 usage hours per month.

The images (virtual environments) were created using the official ROS 1 - noetic - repository on Docker Hub (https://hub.docker.com/r/osrf/ros/tags) on the Ubuntu Focal 20.04 Linux environment.

You can use this platform in two ways:

  • Locally: Download the course and run it on your machine.
  • Online: Use GitHub Codespaces to run the courses directly in the browser, without the need for local installation.

To get started, choose a course and follow the provided instructions. If you have any questions or need assistance, feel free to contact us.

Using LabViR Locally

  • Have a GitHub account - www.github.com
  • Have Docker installed and configured on your machine
  • Have VSCode installed with the Remote Development, ROS, and Docker extensions
  • Your computer can run Windows, Mac, or Linux (Linux is recommended for convenience)

The steps to run courses locally are:

  • Create a folder on your computer to store the courses – e.g., cursos_ros_labvir
  • Access the repository of the desired course via the "GitHub" button. This will clone the repository to your computer.
  • Open VSCode and open the created folder (File -> Add Folder to Workspace).
  • Inside VSCode, click "Remote-Containers: Reopen in Container". This will create a container with all the dependencies needed to run the courses.

Important:

  1. Creating the container may take a few minutes and requires about 6 to 8 GB of disk space.
  2. Once the container is created, the next execution will be much faster.
  • Once the container is open, the virtual environment will be available in your browser. To access it, go to: http://localhost:6080/ and make any necessary screen resolution adjustments.
  • From here, you can open VSCode within the virtual environment and work on the course activities. To do this, add the course folder to your workspace (File -> Add Folder to Workspace), open the README.md file (right-click the tab to preview), and follow the instructions to run the course.

Important:

  1. You can run the course from VSCode in the local environment, but ROS graphical windows will not be displayed. They will only appear in the virtual environment shown in the browser.

Using LabViR Remotely

  • Log in to your GitHub account - www.github.com – we recommend using GitHub Education with a student/teacher account - link
  • Access the course you want to run using the "Open in Codespaces" button. You will be directed to a Codespaces page where you can choose the virtual machine size.
  • If this is your first time using the repository on Codespaces, you will need to create the virtual environment. Click "Create codespace on main" and wait a few minutes for the environment to be created.
  • Once open, Codespaces will launch VSCode in your browser. Just like in the local setup, access the address in the "Ports" tab by clicking "Open in Browser". This will open the virtual environment in your browser. Adjust screen resolution as needed.
  • From here, you can open VSCode within the virtual environment and work normally on the course activities. To do this, add the course folder to your workspace (File -> Add Folder to Workspace), open the README.md file (right-click the tab to preview), and follow the instructions.

Important:

  1. Codespaces is free up to 60 hours per month, so it's important to always close the environment after use.
  2. To close the remote environment, in VSCode click on the Status Bar -> Remote-Containers: Stop Remote Container.
  3. To ensure the environment is closed, go to github.com/codespaces and click "Stop my codespace"
